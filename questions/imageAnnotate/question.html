The questions below were designed to showcase the different features of pl-image-annotate. This element allows students to upload an image and then add rectangular annotations to it that highlight specific areas.

Part 1

Annotation types are pre-defined by placing one or more pl-rectangle-annotate elements inside pl-image-annotate. Each type needs a unique key for internal use and a label that is displayed to students. Only one annotation per type can be placed, and annotations can be place and removed by clicking the corresponding button.
Part 2

Both the annotation element itself and each annotation type can be customized: By setting the width and height attributes in pl-image-annotate, the canvas area can be resized. Note that the canvas height is automatically adjusted to preserve the aspect ratio of uploaded images.

Annotation types can be customized by setting the color, width, height, font-size, and border-width attributes. By default, all annotation rectangles are resizable, but this can be disabled.
Part 3

By default, annotations are not required, but this attribute can be set to automatically mark submissions as invalid if a specific type is missing. This can be a useful setting to simplify auto-grading scripts.

Auto-grading annotated images is not trivial because student-uploaded images such as screenshots or photos might have arbitrary sizes and aspect ratios. The README for this element contains some examples how Optical Character Recognition (OCR) libraries or other tools could be set up to process annotated images automatically.
